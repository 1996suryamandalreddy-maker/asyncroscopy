Python compatibility

The AutoScript client installation bundles AutoScript packages with a Python distribution. This may sometimes conflict with the existing Python environment (different distribution or version) on the user's machine. As a workaround, the client packages can be installed separately in a different Python environment.

Most parts of the AutoScript API will work with any Python version, except for the EmdFile structure. This structure provides functionality to open and modify EMD files in Python and depends on a pyd library that is compiled with a specific Python version. Therefore, this structure is not compatible with a different Python version.

Installing AutoScript packages to a different Python environment

AutoScript packages can be installed using the pip command line tool and a requirements file. Note that you may encounter package compatibility issues related to the current version of the dependencies, pip or python in your environment. We recommend following the steps below:

  • Make sure that the python environment's version is 3.11 or newer. AutoScript client packages don't support older Python versions.
  • Install the AutoScript client with the standard installation executable. This copies all AutoScript packages along with the requirements.txt file to the following directory: 
    C:\Program Files\Thermo Scientific AutoScript TEM\PythonPackages
    The PythonPackages directory can be copied to a different location on the machine or to a different machine. The requirements file contains a complete list of required packages.
  • Open a command line and activate the Python environment where you want to install the AutoScript packages. If you are using conda, it could look something like this: 
    conda activate {environment-name}
  • Utilize pip to install both the packages and their dependencies using the requirements file: 
    pip install -r {path-to-requirements-file}
    For example, to install the AutoScript TEM packages to a Python environment on the same machine, use the following command: 
    pip install -r "C:\Program Files\Thermo Scientific AutoScript TEM\PythonPackages\requirements.txt"

This approach can be utilized to create a hybrid environment with packages of both AutoScript products (AutoScript and AutoScript TEM) installed at the same time. Note that you may encounter issues with displaying these packages in the AutoScript Package Manager in such hybrid environments.

To create such an environment, follow the steps below:

  • First, install both AutoScript products (AutoScript and AutoScript TEM) using the standard installation executables. Make sure you check the option to install the python client packages.
  • Next, to install a product's packages navigate to the directory where the packages are stored, typically:
    • AutoScript TEM: C:\Program Files\Thermo Scientific AutoScript TEM\PythonPackages
    • AutoScript: C:\Program Files\Thermo Scientific AutoScript\PythonPackages
  • After that proceed as described in the previous example to install the packages to your environment.
