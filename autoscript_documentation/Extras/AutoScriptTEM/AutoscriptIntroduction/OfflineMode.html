Offline mode

If the AutoScript server is installed on a non-MPC computer, it starts in offline mode. In this mode, the server operates disconnected from the microscope and forwards all API calls to the built-in TEM simulator. An offline license is required to use AutoScript in offline mode.

An offline AutoScript license cannot be used on an MPC. On an MPC, a non-offline license is required to connect to the microscope.

Offline mode of AutoScript can be installed on any computer that meets the minimum hardware requirements specified in the installation manual. Offline mode supports both local and remote scripting configurations (i.e., clients can connect to the offline server hosted over a network), although the local configuration is simpler and therefore recommended.

Simplistic modeling

The transmission electron microscope is a complex device. The integrated microscope model does not attempt to accurately simulate all of this complexity. Most API functions intended to control the state of the microscope function as a simple read/write cache. A value written by the property setter is stored and returned when the getter is called. This means, for example, that changing the value of beam tilt has no effect on other optical properties.

The state of the simulator is not persisted. When AutoScript server is restarted, the simulator is reset to its initial state and the current state is lost.

Image acquisition

Images output from the detectors or cameras are not visually affected by the state of the microscope or acquisition settings. Static 16-bit images are provided, but in the correct size and format. AutoScript allows you to upload a custom simulation image using the service.offline.set_image() function. This can be particularly useful for simulating parts of the workflow that involve image processing.

Image metadata are simulated as well. Images are created with metadata structures that capture the simulated system state. However, metadata may be incomplete in some areas, e.g. in camera detector settings.

Auto functions

Offline auto functions are not simulated, except for simple input and/or state checks. The result values are filled with zeros.

Offline system properties

  • Maximum acceleration voltage: 300kV.
  • Three condenser lens system, C3 aperture is not motorized.
  • Lorentz mode not supported.
  • Double tilt stage (x, y, z, a, b).
  • Cameras: BM-Ceta, BM-Falcon (Falcon 3 EC), Flucam.
  • STEM available, scanning detectors: BF, DF2, DF4, HAADF, BF-S, DF-S.
  • Stigmators: condenser, diffraction, objective.

System initial state

  • Vacuum state is ready.
  • Specimen holder is set to double tilt.
  • FEG is operational, acceleration voltage: max value.
  • Objective lens mode: HM.
  • Projector mode: Imaging.
  • Magnification: approximately 50k.
