Scanning

In the scanning mode (STEM) of the microscope, images are acquired by moving a focused beam in a raster pattern across the specimen and collecting a signal at each (x,y) pixel coordinate. The signals from all pixels together form the STEM image.

Scan engine

Scan engine is a hardware component that controls the movement and focus of the beam. It synchronizes the detectors and the beam position to ensure the data are acquired at the right time. There are several types of synchronizations:

  • Hardware synchronization: The scan engine is connected to the detector directly.
  • Software synchronization: The scan engine pauses it operation after each point, line or a frame and the cameras are notified via SW.

Size and region

Size defines the outer boundary of the scan area. It is specified in pixels and is limited by the maximum scan size (which is determined by the license). In AutoScript, size can be set with a single integer value that defines the size of the square scan area. The size argument is often optional, and if not set, the default value is the maximum scan size. Region defines a scan pattern that can outline the scan area in a more flexible way. Currently, the only supported region is the rectangular region. All regions must be positioned inside the scan area and can be defined using relative or absolute coordinates.
