About the EMD file format

The EMD file format stands for Electron Microscopy Data and is also known as the Velox experiment file. This file contains not only images, but it is a proprietary container that can contain the following:

  • Acquisition results, such as images, spectras, plots, and maps.
  • Post-processing results.
  • Experiment log.

The EMD file is based on the open format HDF5 (Hiearchic Data Format Version 5). It is possible to use HDF5 software or libraries to read the EMD files (e.g. h5py package in Python). However, it is recommended not to use the libraries to write or modify the EMD files, as this may corrupt the file and make it unreadable by Velox or AutoScript. See snippets for examples of how to read EMD files with the h5py package.

EMD version support

Backward compatibility

The EMD file format evolves with the new versions of Velox and has its own version. Backward compatibility is achieved by converting the files to a newer version. Velox does this automatically by saving the old file version in a new format. In AutoScript, a conversion function must be called explicitly to convert the file to the new EMD version.

EMD file format version used by this version of AutoScript is 4.3.

Forward compatibility

Forward compatibility is not supported. This means that older versions of AutoScript and Velox will not be able to open files that were saved in the newer versions of both applications.

AutoScript and Velox release cycles

AutoScript and Velox release cycles are synced, and the newest version of AutoScript will always support the files saved by the newest version of Velox and vice-versa. The newest versions of both Velox and AutoScript are available here.
