AdornedImage.save

Saves the image to the specified path. Supported formats: tiff, png, bmp, gif.

Syntax

save(path)

Parameters

path Path of the file to be saved.

Remarks

The path extension determines which image format to use. If the file has no extension, TIFF is used by default. An error will be raised if an unsupported extension is used. TIFF is a preferred image format because it allows metadata to be stored with the image.

If BMP or GIF format is used, the image is first normalized and converted to 8-bit RGB format (if not already). If the PNG format is used, 32-bit images are converted to 16-bit when saved (with data loss!). These operations have no effect on the objects in memory, only on the images stored on disk.

TIFF is considered a raw data format, meaning no automatic corrections are applied to the image. If the image contrast is too low, you must explicitly call the stretch_histogram function before saving the image.

Examples

The following example shows different ways to save the image to disk:


        # Load an image from disk
        image = AdornedImage.load(r"c:\image.tiff")

        # Resave the image in tiff format
        image.save(r"c:\AutoScript images\image")
        image.save(r"c:\AutoScript images\image2.tiff")

        # Save the image in png format
        image.save(r"c:\AutoScript images\image.png")

        # Save the image in bitmap format
        image.save(r"c:\AutoScript images\image.bmp")

        # Save the image in gif format
        image.save(r"c:\AutoScript images\image.gif")
