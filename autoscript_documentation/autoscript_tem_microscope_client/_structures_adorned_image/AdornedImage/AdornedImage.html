AdornedImage

Contains microscope image data along with metadata describing the state of the microscope under which the image was acquired.

Items

width [Read only] int The width of the image in pixels.
height [Read only] int The height of the image in pixels.
bit_depth [Read only] int Bit depth of the image data.
metadata [Optional] AdornedImageMetadata Metadata containing information about the system setup at the time the image was captured.
data [Read only] Numpy.ndarray The data arranged either in 2D (for grayscale images) or 3D (for RGB images) numpy array.
encoding [Read only] int Encoding of the image. To explain this property use enumeration ImageDataEncoding.
checksum [Read only] int The checksum of the raw data.
thumbnail [Read only] AdornedImage The thumbnail of the original image.

Methods

load() Loads an image from a file. Supported formats: tiff, png, bmp.
save() Saves the image to the specified path. Supported formats: tiff, png, bmp, gif.

Remarks

The structure contains both the numpy array representing the image and the metadata. It is used as a result by most acquisition functions, but can also be created manually by specifying a numpy array and a metadata object (both arguments are optional). The object construction uses deep copy internally for the metadata object to avoid overwriting them later. This way, multiple images do not share the same metadata object.

Metadata

When AutoScript loads a file from disk using the load() function, the existing EXIF metadata is lost during the subsequent save(). AutoScript uses its own metadata structure, which is employed by most Thermo Fisher applications; this structure is defined in AdornedImageMetadata.

Supported formats

The following table explains the supported bit depths and encoding combinations. Note that this table refers to the default and recommended TIFF image format. Other formats may contain additional unsupported combinations. See save and load pages for more information.

UNSIGNED SIGNED RGB BGR
8 Supported - - -
16 Supported Partially supported * - -
24 - - Supported (8 bits per channel) Supported (8 bits per channel)
32 - Supported - -

* AutoScript does not support save and load operations with signed 16-bit grayscale images. As a workaround, these images are converted to int32.

Examples

The following example shows how to calculate the mean of the image (using the numpy package):


        import numpy as np
        print(np.mean(image.data))

The next example shows how to load a TIFF file from the disk:


        image = AdornedImage.load("c:\\image.tiff")

The next example shows how to create an empty image with a numpy array and a metadata object:


        # Creates a numpy array filled with zeroes
        numpy_array = numpy.full(shape=(1024, 1024), fill_value=0, dtype=numpy.uint16)

        # Creates a blank metadata object
        metadata = AdornedImageMetadata()

        # Creates an empty image
        image = AdornedImage(numpy_array, metadata)

        # Created metadata object and the image metadata object are different 
        # (a deep copy was performed to avoid overwriting data) 
        assert id(metadata) != id(image.metadata)
