Dynamic objects

Dynamic objects are objects that live on the server and have only a thin representation on the client. This means that the client can work with the dynamic object as if it were a simple data structure, but all calls are resolved by the server. Dynamic objects are often used to model and encapsulate more complex functionality (e.g. detectors, aperture mechanisms).

AdornedSpectrumImage Handle to the spectrum image result existing on the server.
ApertureMechanism Contains properties and enables control of an aperture mechanism.
BiPrism Contains properties and enables control of a bi-prism.
BiPrismRotation Contains properties and enables control of a bi-prism rotation.
BiPrismVoltage Contains properties and enables control of a bi-prism voltage.
CameraDetector Contains properties of a camera detector.
DetectorSegment Represents a single segment on a ScanningDetector.
EdsDetector Contains properties and enables control of a EDS detector.
EdsDetectorSegment Represents a single segment on a EdsDetector.
EelsDetector Contains properties of an EELS detector.
Lens Contains properties of a lens.
PresetRaw PresetRaw contains properties for setting and getting the preset value and the range of the preset.
ScanningDetector Contains properties and enables control of a scanning (STEM) detector.
XLensAlignment Contains properties of an X lens alignment.
