acquisition.acquire_stem_data_advanced

Triggers a scan task without returning the data.

Syntax

acquire_stem_data_advanced(settings)

Parameters

settings StemDataSettings Advanced settings structure.

Remarks

The function extends the acquire_stem_data function by allowing the user to control the acquisition in more detail. The data is not returned, but stored on a different computer (offloaded), depending on the camera type used:

  • BM_EMPAD or SH_EMPAD:

    The resulting data are stored on the EMPAD PC. Size of the image from each scan point is 128x128 and cannot be changed. For experiments with EMPAD, it is recommended to upgrade to the latest available version of the TEM server software and the EMPAD control software. Older versions are known to have issues with storing acquired images on the EMPAD computer.

  • BM_CETA:

    The resulting data are stored on the storage server. Size of the image from each scan point is 256x256 and cannot be changed.

  • Other detector types are not supported.
  • If the detector type is not provided, the function will run an "empty" scan that can be used for third-party detector acquisition. AutoScript cannot access the data, and the detector must be hardware synchronized in this case.

Keep in mind that after the function returns, it may take some time before the data is actually stored and can be accessed. The time depends on the size of the resulting data.

Examples

The following example shows how to trigger a 4D STEM acquisition with the EMPAD detector:


        # Change optical mode to STEM
        microscope.optics.optical_mode = OpticalMode.STEM

        # Trigger EMPAD scan
        microscope.acquisition.acquire_stem_data_advanced(StemDataSettings(dwell_time=1e-3, size=ImageSize.PRESET_128, detector_types=[CameraType.BM_EMPAD]))

The next example shows how to trigger a 4D STEM acquisition with the Ceta detector:


        # Change optical mode to STEM
        microscope.optics.optical_mode = OpticalMode.STEM

        # Trigger scan
        microscope.acquisition.acquire_stem_data_advanced(StemDataSettings(dwell_time=10e-3, size=ImageSize.PRESET_128, detector_types=[CameraType.BM_CETA]))

The next example show how to trigger acquisition with a region and a 3rd party detector:


        # Change optical mode to STEM
        microscope.optics.optical_mode = OpticalMode.STEM

        # Create settings structure
        settings = StemDataSettings(dwell_time=1e-3, size=ImageSize.PRESET_512,
                                    region=Region(RegionCoordinateSystem.PIXELS, rectangle=(0, 0, 30, 30)))

        # Trigger scan
        microscope.acquisition.acquire_stem_data_advanced(settings)
