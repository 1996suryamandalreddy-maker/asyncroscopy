acquisition.acquire_stem_data

Triggers a scan task with a given scan synchronizable camera without returning the data.

Syntax

acquire_stem_data(camera_detector, size, dwell_time)

Parameters

camera_detector str The name of the camera detector. Enumeration CameraType can be used.
size int Square scan size in pixels. Enumeration ImageSize can be used.
dwell_time float The dwell time in seconds.

Remarks

The function can be used to trigger acquisitions with scan synchronized cameras to acquire one camera image per scan point. The data is not returned, but stored on a different computer (offloaded), depending on the camera type used:

  • BM_EMPAD or SH_EMPAD:

    The resulting data are stored on the EMPAD PC. Size of the image from each scan point is 128x128 and cannot be changed. For experiments with EMPAD, it is recommended to upgrade to the latest available version of the TEM server software and the EMPAD control software. Older versions are known to have issues with storing acquired images on the EMPAD computer.

  • BM_CETA:

    The resulting data are stored on the storage server. Size of the image from each scan point is 256x256 and cannot be changed.

  • Other detector types are not supported.

Keep in mind that after the function returns, it may take some time before the data is actually stored and can be accessed. The time depends on the size of the resulting data.

Examples

The following example shows how to trigger a 4D STEM acquisition with the EMPAD detector:


        # Change optical mode to STEM
        microscope.optics.optical_mode = OpticalMode.STEM

        # Trigger EMPAD scan, scan size: 128x128, dwell time 1ms
        microscope.acquisition.acquire_stem_data(CameraType.BM_EMPAD, ImageSize.PRESET_128, 1e-3)

The next example shows how to trigger a 4D STEM acquisition with Ceta detector:


        # Change optical mode to STEM
        microscope.optics.optical_mode = OpticalMode.STEM

        # Trigger Ceta scan, scan size: 128x128, dwell time 10ms
        microscope.acquisition.acquire_stem_data(CameraType.BM_CETA, ImageSize.PRESET_128, 10e-3)
