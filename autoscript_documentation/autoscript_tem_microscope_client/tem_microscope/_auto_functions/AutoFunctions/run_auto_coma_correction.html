auto_functions.run_auto_coma_correction

Performs automatic coma correction. Available only in TEM mode.

Syntax

run_auto_coma_correction(settings)

Parameters

settings RunAutoComaCorrectionSettings Settings for the auto coma routine.

Return value

RunAutoComaCorrectionResults A structure capturing results of the autofunction run.

Remarks

The autofunction consists of two steps:

  • Measure coma by fitting Thon rings of a beam tilt tableau.
  • Automatically correct for measured coma error.

Most of the parameters in RunAutoComaCorrectionSettings have default values and can be omitted. The first three parameters are linked to image acquisition and have the same meaning as the arguments to the acquire_camera_image() function. The default values are documented on the RunAutoComaCorrectionSettings page.

For information on supported camera types see members of the CameraType enumeration. Running this autofunction with the FLUCAM camera type is not supported.

The screen does not need to be retracted before calling this auto function; the acquisition part of the task manages it automatically. The selected detector is also inserted automatically.

Preconditions and recommendations:

  • Only available on systems without Image Corrector.
  • optical_mode must be TEM.
  • projector_mode must be IMAGING.
  • The microscope is properly aligned. Check the Direct Alignments (especially rotation center) and roughly correct objective astigmatism.
  • EPU / Auto Functions / Autofocus calibration has been performed.
  • Magnification calibrations have been performed.
  • A specimen is loaded which enables the display of Thon rings in the power spectrum of the TEM image. This is typically a thin amorphous carbon film, such as Quantifoil, Lacey Carbon, or a Cross Grating sample.
  • Spot size is 3 or higher.
  • Defocus is between -1 µm and -3 µm.
  • Magnification is approximately 150 kX.
  • The beam diameter can be narrowed a bit to increase signal strength.
  • Accurately set the specimen to eucentric height, using the Stage Alpha tilt wobbler.

If the coma error cannot be corrected within a specified number of iterations, the optical settings are reset to the state before the start of the correction procedure.

Examples

The following example shows how to run the auto coma correction routine with default parameters:


        # Save settings into a variable with all optional settings left as defaults
        settings = RunAutoComaCorrectionSettings(CameraType.BM_CETA)
        # Run the auto coma correction routine with the created settings
        result = microscope.auto_functions.run_auto_coma_correction(settings)

Optional parameters can be added to the structure: 


        settings = RunAutoComaCorrectionSettings(CameraType.BM_CETA, 1024, 2, 5, tile_size=1024)
        microscope.auto_functions.run_auto_coma_correction(settings)
