analysis.eels.acquire_spectrum_image

Acquires EELS spectrum image.

Syntax

acquire_spectrum_image(settings)

Parameters

settings EelsSpectrumImageSettings EELS spectrum image settings structure.

Return value

AdornedSpectrumImage Object handle that provides a way to download the acquired data to the client.

Remarks

Spectrum image acquisition is not available in offline mode.

File representation

The acquired handle is associated with a file on the MPC disk. The handle is only valid for a limited time. During the AutoScript startup sequence, the acquired files are checked, and all files older than 24 hours are deleted.

The save() method can be used to download the acquired data to the client. This method saves the spectrum image in a Velox EMD format. Information on the compatibility of the EMD file format can be found on the Extras/EMD file format page in the reference manual.

File contents

The data is hierarchically structured and uses the binary data format HDF5. The Python package h5py can be used to read the file directly. Alternatively, the Python package hyperspy or the Velox application can be used to read the file and/or further analyze the data.

The data is organized as follows:

  • EELS spectrum image + Acquisition metadata + Metadata

    • Acquired spectra are saved into a cube with dimensions of (width x bin count x height). There is one cube per exposure setting. Acquisition metadata is a simple JSON object with information about the acquisition (dispersion, offset, etc.). There is one acquisition metadata object per scan line of the image (eg. image with dimensions of 10x10 will have 10 acquisition metadata objects). Metadata have the same structrure as AdornedImageMetadata but are stored in JSON format. There is one metadata object per cube.

  • STEM images + metadata

    • All STEM frames are stored with the corresponding metadata. Metadata have the same structrure as AdornedImageMetadata but are stored in JSON format.

    A STEM detector can be specified either through a detector type (using scanning_detectors parameter) or through a list of segments (using scanning_detector_segments parameter). While it is possible to combine these two settings, it's important to note that a detector cannot be simultaneously defined by both its segment and its type. Each detector and segment individually capture an image.

Performance and limitations

This iteration of the SI acquisition is considered to be a minimum viable product. We will continue to iterate on it and add more features in the future. If you have any suggestions or requests, please let us know (see the support page for contact info).

To minimize acquisition time, it is recommended to restrict the scan region to a maximum of 512x512 pixels. Keep in mind that the stability of the acquisition may not be optimal for larger regions. Data loss cancels the data saving in order to avoid corrupted files. If such a situation occurs, the acquisition will be stopped and an error will be raised. The function spectrum_image_history can be used to retrieve the incomplete file.

The maximum number of exposure settings is detector dependent. The dwell time must be higher than the sum of exposure times (plus the necessary overhead depending on the number of spectra and HW limitations).

Examples

The following example shows EELS spectrum image acquisition with Zebra and HAADF detectors:


        # Configure the EELS spectrum image acquisition
        # The result will contain 3 spectrum cubes (one for each exposure setting) and 1 STEM image (HAADF)
        settings = EelsSpectrumImageSettings()
        settings.eels_detector = EelsDetectorType.ZEBRA
        settings.eels_exposure_settings = [
            EelsExposureSettings(exposure_time=10e-3, drift_tube_energy_offset=20),
            EelsExposureSettings(exposure_time=10e-3, drift_tube_energy_offset=40),
            EelsExposureSettings(exposure_time=10e-3, drift_tube_energy_offset=60)]
        settings.dwell_time = 35e-3
        settings.size = ImageSize.PRESET_512
        settings.scanning_detectors = [DetectorType.HAADF]

        # Acquire the spectrum image
        eels_spectrum_image = microscope.analysis.eels.acquire_spectrum_image(settings)

        # Save the spectrum image to disk (the object is just a handle for the data, we need to transfer it to the client)
        eels_spectrum_image.save(r"C:\temp\spectrum_image.emd")

The next example shows the use of STEM detector segments (two detectors combined) and a rectangular region:


        # Get the BF-S detector and select inner ring segments
        bf_s = microscope.detectors.get_scanning_detector(DetectorType.BF_S)
        bf_s.set_enabled_segments(DetectorSegmentType.INNER_RING)

        # Get the DF-S detector and select outer ring segments
        df_s = microscope.detectors.get_scanning_detector(DetectorType.DF_S)
        df_s.set_enabled_segments(DetectorSegmentType.OUTER_RING)

        # Combine enabled segments
        segments = df_s.get_enabled_segments() + bf_s.get_enabled_segments()

        # Configure the rectangle region (75x100 pixels)
        region = Region(coordinate_system=RegionCoordinateSystem.PIXELS, rectangle=[0, 0, 75, 100])

        # Configure the EELS spectrum image acquisition
        settings = EelsSpectrumImageSettings()
        settings.eels_detector = EelsDetectorType.ZEBRA
        settings.eels_exposure_settings = [EelsExposureSettings(exposure_time=10e-3, drift_tube_energy_offset=20)]
        settings.dwell_time = 20e-3
        #settings.region = region
        settings.scanning_detector_segments = segments

        # Acquire the spectrum image - the resulting file contains 8 STEM images (1 per segment of the two detectors)
        # Each STEM image has a size of 75x100 pixels
        eels_spectrum_image = microscope.analysis.eels.acquire_spectrum_image(settings)
