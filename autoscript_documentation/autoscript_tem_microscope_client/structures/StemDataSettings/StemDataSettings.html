StemDataSettings

Contains advanced settings for scan trigger acquisition.

Items

dwell_time float STEM dwell time in seconds.
detector_types [Optional] List[str] The name(s) of the detector(s) used for the acquisition. Enumeration CameraType can be used.
region [Optional] Region Acquisition area to be scanned. Use Region.
size [Optional] int Square scan size in pixels. Enumeration ImageSize can be used.

Remarks

This structure contains a set of settings used for the advanced stem data acquisitions, where the data are not handled by AutoScript. Not all settings are intercompatible. A valid Settings structure must adhere to the following criteria:

  • dwell_time must be specified.
  • Either size or region must be specified. If both are specified, the region must fit into the size dimensions. If the size is not specified, the maximum scan size of the system is assumed.
  • region has to contain exactly one shape.
  • detector_types list can be empty or None (in that case, any 3rd party detector can be connected to the scan engine directly and HW synchronization is used). If a detector is specified, it must be one of the EMPAD-type cameras. The list can contain multiple detectors.

When a region is specified, both the scanning area and scanning time are reduced. This technique can be utilized for optimization purposes. Additionally, the resulting images will have a smaller size, as they will be cropped to match the specified region area.
